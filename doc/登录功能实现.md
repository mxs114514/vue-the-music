核心逻辑设计：

注册流程：前端提交用户名/密码 -> 后端接收 -> 加密密码 (不能存明文) -> 存入数据库 -> 返回成功。
登录流程：前端提交用户名/密码 -> 后端查找用户 -> 比对密码 (将提交的密码加密后与数据库比对) -> 比对成功 -> 生成 Token (JWT) -> 返回 Token 给前端。
前端状态管理：前端拿到 Token -> 存入 Pinia (并持久化到 localStorage) -> 以后每次请求带上 Token -> 路由跳转到主页。
【制定方案】
我将分四个阶段来指导你完成这个功能。

阶段一：后端基础设施准备 (Server)

目标：让后端具备加密和签发“身份证”(Token) 的能力。
操作：安装 bcryptjs (加密) 和 jsonwebtoken (JWT)。
阶段二：后端业务逻辑实现 (Server)

目标：实现注册和登录的 API 接口。
操作：
创建 user.controller.ts：编写注册（Hash密码、存库）和登录（比对密码、生成Token）的逻辑。
创建 user.routes.ts：定义 /api/auth/register 和 /api/auth/login 路由。
在 index.ts 中注册路由。
阶段三：前端状态管理与网络层 (Client)

目标：前端能够发起请求，并记住“我登录了”。
操作：
安装 axios (推荐，比 fetch 好用)。
创建 src/utils/request.ts (可选，封装请求)。
创建 src/stores/auth.ts：定义 login 和 register action，管理 token 和 userInfo。
阶段四：前端界面实现 (Client)

目标：画出登录/注册页面，并把所有逻辑串起来。
操作：
编写 LoginView.vue：使用 Element Plus 表单。
修改 index.ts：登录成功后跳转。
